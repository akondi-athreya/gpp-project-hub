### `01` 🔐 Cipher-Dock — *PKI-Based 2FA Microservice*
**[`→ View Repository`](https://github.com/akondi-athreya/gpp-1-cipher-dock)**

```
DOMAIN   │ Security · Cryptography · Backend
STACK    │ Node.js · Express.js · Docker · RSA-2048 · TOTP (RFC 6238)
PATTERN  │ Microservice · Multi-Stage Docker Build · Cron Jobs
```

Enterprise-grade cryptographic microservice that fuses **RSA asymmetric encryption** with **RFC 6238 TOTP** into a single containerized auth service. It accepts an RSA-encrypted seed via REST, decrypts it using a 2048-bit private key with OAEP+SHA-256 padding, stores it in a Docker volume, and fires 6-digit time-based OTPs with a 30-second validity window.

Background cron workers run continuously — a heartbeat every 30 seconds and a 2FA audit logger every 60 seconds — simulating production-grade compliance infrastructure.

```
✦ RSA-2048 + OAEP/SHA-256 — enterprise seed protection
✦ ±30s clock-skew tolerance for replay attack prevention
✦ Multi-stage Docker build → minimal production footprint
✦ Cron-based audit logging baked into the container
```

### `11` 🔄 Data Sync Pipeline — *Debezium · Kafka · CDC*
**[`→ View Repository`](https://github.com/akondi-athreya/gpp_data_synchronization_pipeline)**

```
DOMAIN   │ Data Engineering · Event-Driven Architecture
STACK    │ Node.js · Apache Kafka · Debezium · PostgreSQL · MongoDB
PATTERN  │ CDC (Change Data Capture) · CQRS · LSN-Based Idempotency
```

Real-time **Change Data Capture** architecture using the industry-standard Debezium + Kafka stack to keep a PostgreSQL write model and MongoDB read model in sync — without polling.

The `write-service` exposes CRUD APIs for a `products` table. Debezium captures **PostgreSQL WAL** changes and streams them as Kafka events. The `read-service` consumes these events and projects into MongoDB as a queryable read model. LSN-based idempotency prevents duplicate processing. 7-service Docker Compose stack with automated smoke testing.

```
✦ Debezium CDC captures real WAL changes — zero polling, zero loss
✦ LSN-based idempotency — safe consumer restart
✦ CQRS split: write → PostgreSQL · read → MongoDB
✦ Full 7-service Docker Compose with smoke.sh verification
```

### `14` 🤖 LLM Prompt Router — *Intent Classification + Persona Routing*
**[`→ View Repository`](https://github.com/akondi-athreya/gpp-llm-powered-prompt-router)**

```
DOMAIN   │ AI · LLM Architecture · Production AI Systems
STACK    │ Python · FastAPI · OpenAI-Compatible API · Docker
PATTERN  │ Two-Stage LLM Pipeline · Confidence Thresholding · JSONL Audit Log
```

Two-stage AI architecture used in real production systems: a **Classification Stage** detects user intent via a compact LLM call returning structured JSON, then a **Generation Stage** routes to a specialist persona — Code Expert, Data Analyst, Writing Coach, or Career Advisor — for a focused, high-quality response.

If classifier confidence falls below a configurable threshold, the system asks a clarifying question instead of guessing. Manual overrides (`@code`, `@data`) bypass the classifier. Every request logged to `route_log.jsonl`. OpenAI-compatible — works with any provider (OpenAI, Groq, etc.) by setting `OPENAI_BASE_URL`.

```
✦ Two-stage LLM: classify intent first → generate with right persona
✦ Configurable confidence threshold with intelligent fallback
✦ Manual @override support for power users
✦ JSONL audit log: intent · confidence · message · response
```
